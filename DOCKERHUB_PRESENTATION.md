# EGAIS Normatives Assistant (RAG)

Локальный AI-ассистент по нормативам ЕГАИС и лицензированию алкогольной продукции.  
Фокус: **точные ответы с проверяемыми источниками**, режимы **User/Expert**, и контроль качества через eval + регрессию тестов.

---

## Что это дает бизнесу

| Задача | Как решено | Практический эффект |
|---|---|---|
| Быстро найти норму в НПА | TF-IDF retrieval + embeddings re-rank | Меньше времени на поиск релевантных документов |
| Снизить риск "выдуманных" ответов | Source sanitizer + strict source reconstruction + guardrails | Выше доверие к ответу и источникам |
| Дать понятный ответ не-эксперту | User mode со структурой "что сделать / какие документы / что уточнить" | Легче использовать в рабочих сценариях |
| Оставить гибкость для аналитика | Expert mode с расширенными настройками retrieval/LLM | Точная отладка и углубленная проверка |

---

## Ключевые возможности

- Retrieval pipeline: lexical поиск -> embeddings re-rank -> multi-step retrieval -> hierarchy expansion.
- Ответы с источниками и проверкой факт-консистентности.
- Обработка запросов "нужен список документов": если список найден в контексте - возвращается явный перечень; если нет - явно сообщается об отсутствии информации.
- FAQ-панель в UI с быстрыми типовыми вопросами.
- Диагностика и логи для анализа качества и UX.

## Архитектура (кратко)

| Слой | Компоненты |
|---|---|
| Данные | Нормализованный корпус НПА, чанки, индексы |
| Retrieval | TF-IDF index, embeddings re-rank, multi-step queries |
| Generation | LLM backend (локально/облако), structured answer formatting |
| Quality & Safety | Guardrails, source sanitizer, confidence signals |
| UI | Gradio, User/Expert режимы, FAQ, настройки |

---

## Метрики качества

| Метрика | Последнее значение | Как проверяется |
|---|---|---|
| Eval `extra10` | `ok=10/10` | Автопрогон сценариев и проверка итогов |
| Regression tests | `65 passed` | `pytest` для critical guards + formatting |
| Источники | Снижены ошибки ссылок | Санитайзер + реконструкция источников |
| Галлюцинации | Снижены на практических кейсах | Пост-валидация и rule-based guards |
